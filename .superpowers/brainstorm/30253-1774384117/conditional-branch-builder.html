Conditional Branch Builder

Revised recommendation: one Conditional node becomes an ordered branch evaluator with nested boolean groups inside each branch.

Recommended

If / Else If / Else with Nested Groups

Authors build readable branches instead of a flat bag of rules. Each branch has its own condition tree and output path. The node evaluates branches top to bottom and takes the first matching branch, with a final default else path.

Why

  • Matches how people already think about automation logic.
  • Supports nesting without requiring a separate mini-language.
  • Scales to multiple outcomes: success path, retry path, escalation path, default path.

Tradeoff

  • Slightly richer data model than a single predicate, but still deterministic and efficient.

5. Data Model Recommendation

A

Ordered branches

Conditional config stores a top-down branch list with exactly one optional else.

B

Nested expression tree per branch

Each branch condition is a typed tree of groups and leaves, not flat text.

C

Typed workflow globals

One globals node owns workflow defaults, runtime-override policy, and task mutation eligibility.
