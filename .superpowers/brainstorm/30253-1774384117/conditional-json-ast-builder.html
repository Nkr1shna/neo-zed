Language-Agnostic Conditional Builder

Recommended production design: block-based nested branch editor with a typed JSON AST backing model and a deterministic evaluator.

Recommended

Structured Blocks + JSON AST

Authors build nested logic entirely through UI controls. The saved workflow stores a typed JSON tree. Runtime uses one evaluator that walks the tree and returns the selected branch output.

Rejected

No Embedded DSL

No text box for writing conditions. No language-specific syntax. No parser edge cases in authoring.

Approach Comparison

A

Flat rule list

Simple, but fails your nested branching requirement.

B

Text expression DSL

Compact, but too opaque, language-like, and error-prone for production UX.

C

Nested block builder with JSON AST

Best fit: unlimited nesting, stable validation, portable execution, and safe UI authoring.
